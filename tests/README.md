# E2E тесты для FreeDiary

End-to-end тесты на Playwright для проекта FreeDiary (FastAPI + React).

## Требования

- Node.js 16+
- Python 3.8+ (для бэкенда)
- Запущенные серверы:
  - Фронтенд: http://localhost:3000
  - Бэкенд: http://localhost:8000

## Установка

1. Установите зависимости Playwright:
```bash
npm install
```

2. Установите браузеры для Playwright:
```bash
npm run playwright:install
```

Или установите конкретные браузеры:
```bash
npm run playwright:install-chromium
npm run playwright:install-firefox
npm run playwright:install-webkit
```

## Запуск тестов

### Все тесты
```bash
npm run test:e2e
```

### Тесты с UI-режимом
```bash
npm run test:e2e:ui
```

### Тесты в режиме отладки
```bash
npm run test:e2e:debug
```

### Тесты для конкретного браузера
```bash
npm run test:e2e:chromium
npm run test:e2e:firefox
npm run test:e2e:webkit
```

### Генерация HTML-отчета
```bash
npm run test:e2e:report
```

## Структура тестов

- `auth.spec.ts` - тесты авторизации и регистрации
- `trainings.spec.ts` - тесты управления тренировками
- `stats.spec.ts` - тесты страницы статистики
- `telegram.spec.ts` - тесты интеграции с Telegram

## Сценарии тестирования

### Авторизация и регистрация
1. Регистрация нового пользователя
2. Вход с существующими учетными данными
3. Вход с неверными данными
4. Выход из системы

### Управление тренировками
1. Создание новой тренировки (тип: Бассейн)
2. Валидация формы (пустое название)
3. Доступ без авторизации
4. Просмотр деталей тренировки с упражнениями
5. Удаление тренировки

### Статистика
1. Просмотр страницы статистики
2. Работа с фильтрами
3. Доступ без авторизации
4. Детальная статистика по типам тренировок

### Telegram интеграция
1. Подключение Telegram аккаунта
2. Отправка тестового уведомления
3. Отключение Telegram
4. Просмотр статуса подключения
5. Обработка ошибок подключения

## Настройка окружения

### Пропуск тестов Telegram
Если Telegram API недоступен, установите переменную окружения:
```bash
export SKIP_TELEGRAM_TESTS=true
```

Или используйте моки (уже реализованы в тестах).

### Конфигурация Playwright
Настройки находятся в `playwright.config.ts`:
- Базовый URL: http://localhost:3000
- Автозапуск серверов фронтенда и бэкенда
- Поддержка Chrome, Firefox, Safari
- HTML-репортер

## Очистка данных

Тесты автоматически очищают созданные данные через API:
- После регистрации - удаление тестового пользователя
- После создания тренировки - удаление тренировки
- Используются уникальные данные для каждого теста

## Рекомендации

1. **Перед запуском тестов:**
   - Убедитесь, что серверы запущены
   - Запустите seed-данные: `cd backend && python seed.py`
   - Проверьте подключение к базе данных

2. **Для отладки:**
   - Используйте `npm run test:e2e:debug` для пошагового выполнения
   - Используйте `npm run test:e2e:ui` для визуального отладчика
   - Просматривайте трассировки в `test-results/`

3. **Для CI/CD:**
   - Установите переменную окружения `CI=true`
   - Используйте `--headed` для headless-режима
   - Настройте репортеры в `playwright.config.ts`

## Устранение неполадок

### Ошибки сети
- Проверьте, что серверы запущены на правильных портах
- Убедитесь, что нет блокировок firewall

### Ошибки авторизации
- Проверьте seed-данные в базе
- Убедитесь, что пароли хэшируются корректно

### Ошибки Playwright
- Переустановите браузеры: `npm run playwright:install`
- Обновите Playwright: `npm update @playwright/test`

## Дополнительные тесты

2. Импортируйте `test` и `expect` из `@playwright/test`

4. Обеспечьте очистку созданных данных
5. Добавьте описание теста на русском языке
